# Bak-Man Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans (inline) to implement task-by-task. Steps use checkbox (`- [ ]`) syntax. Logic tasks are TDD with Vitest; rendering/scene/audio tasks are verified live via the Vite dev server + browser preview.

**Goal:** Build a mobile-first, Pac-Man-style maze arcade game ("Bak-Man") starring Bak the dog vs. household vacuums, playable in mobile Safari and deployable as a PWA.

**Architecture:** Fixed-timestep loop drives a tile-based maze model. Entities (Bak, 4 vacuum types, boss) move grid-locked between tile centers. Pure-function systems (AI targeting, collision, scoring) are unit-tested; rendering reads game state and draws to a single Canvas 2D context. A scene/state manager sequences title â†’ rooms â†’ boss â†’ win/gameover. Real sprites load via an asset manifest with procedural fallbacks.

**Tech Stack:** TypeScript, Vite, Vitest (+ jsdom), vite-plugin-pwa, HTML5 Canvas 2D, Web Audio API. No game framework.

## Global Constraints

- **Logical playfield is portrait**, designed around a tile grid; the engine scales the grid to fit the device with letterboxing and respects safe-area insets. Target 60fps.
- **Playable with placeholder art at all times** â€” every sprite has a procedural fallback; missing files never crash or block.
- **Sprites load from `assets/sprites/`** by the exact names in `docs/ASSETS.md`.
- **Palette & type from `docs/THEME.md`** (Bak-orange accent `#ff7a1a`; per-vacuum hues; per-room wall colors; Fredoka/Nunito).
- **No copyrighted/trademarked assets or names.** Original engine + synth audio.
- **Commits:** frequent, conventional-commit style. (Git is initialized in Task 1 only when the user approves committing; otherwise work stays uncommitted.)
